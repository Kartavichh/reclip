# 03. Демон (фоновое слежение)

Решения по группе 3 из `TODO.md`.

## 3.1 Интервал поллинга — ✅ 500 мс
Демон заглядывает в буфер каждые **500 мс** по умолчанию. Незаметно для человека,
нагрузка near-zero. Значение позже вынесем в настройки (группа 6).

## 3.2 Как запускается — ✅ вручную (разработка) + systemd user-сервис (боевой)
- На этапе разработки демон запускаем **руками в терминале** — удобно смотреть логи.
- Для реального использования кладём в проект **systemd user-сервис**
  (`reclip.service`): авто-старт при входе, авто-перезапуск при падении,
  управление через `systemctl --user`.

## 3.3 Один экземпляр — ✅ файл-замок (flock)
Демон при старте берёт эксклюзивный `flock` на
`$XDG_RUNTIME_DIR/reclip.lock`. Если замок занят — второй экземпляр вежливо
завершается (чтобы два демона не дрались за буфер).

## 3.4 Способ чтения буфера — ✅ `arboard` за trait, с ранней проверкой на GNOME
- **Основной вариант:** крейт **`arboard`** (читает и пишет; X11/Wayland/Windows;
  без внешних программ; ложится в будущую кроссплатформу).
- Прячем его за trait-«переходник» (см. 7.2), чтобы реализацию можно было
  заменить, не трогая остальной код.
- **Риск и план Б:** GNOME/Wayland капризен. Поэтому в **самом первом кодовом
  шаге (Этап 0)** обязательно проверяем `arboard` на реальном GNOME. Если не
  заработает — переключаемся на `wl-clipboard-rs` или вызов `xclip` (всё за
  тем же trait, без переделки логики).

> Связано с 7.2 (форма trait-абстракции) и с проблемой «эха» (группа 5).
