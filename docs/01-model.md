## 1.1 Типы содержимого в MVP — ✅ только текст
В MVP храним **только текст**. Но в модели сразу закладываем «тип записи»
(перечисление), чтобы картинки добавлялись отдельной веткой без переделки:

```rust
// иллюстрация, не финальная форма
enum Content {
    Text(String),
    // Image(...) — добавим позже, не ломая остальной код
}
```

## 1.2 За каким буфером следим — ✅ только CLIPBOARD
В Linux два выделения: `CLIPBOARD` (`Ctrl+C`) и `PRIMARY` (выделение мышью).
PRIMARY меняется при каждом выделении и засоряет историю.
**MVP следит только за `CLIPBOARD`.** Поддержку PRIMARY отложили.

## 1.3 Дубликаты — ✅ поднимать наверх
Если копируемое содержимое уже есть в истории — **не создаём новую запись, а
перемещаем существующую наверх** (поведение как у `Win+V`). Сравнение — по
содержимому.

## 1.4 Пустое/мусорное — ✅ игнорировать
**Не сохраняем** пустой буфер и строки, состоящие только из пробелов/переводов
строк. (Сам буфер при этом не трогаем, речь только про запись в историю.)

## 1.5 Лимит размера одной записи — ✅ порог ~1 МБ, не сохранять сверх
Если содержимое больше порога (по умолчанию **1 МБ**) — **не сохраняем его в
историю** (буфер продолжает работать штатно). Обрезать нельзя — испортит вставку.
Порог позже вынесем в настройки.

## 1.6 Сколько записей храним — ✅ по умолчанию 100, вытесняем старые
Максимум **100** последних записей. При переполнении удаляем самые старые
(FIFO). Число позже сделаем настраиваемым. На будущее: закреплённые (pin)
записи вытесняться не должны.
