## 6.1 Как ставить — ✅ сборка из исходников + `install.sh`
MVP: `cargo build --release`, затем **`install.sh`**, который:
- копирует бинарь в `~/.local/bin/reclip`;
- ставит systemd user-сервис (`reclip.service`) и включает его;
- печатает инструкцию по настройке `Super+V` в GNOME.

Готовые бинари в GitHub-релизах (без сборки) — цель на будущее, требует CI.

## 6.2 Системные зависимости — ✅ проверяет install.sh
- **Для сборки:** `build-essential` (C-компилятор для SQLite/GTK), `libgtk-4-dev`
  и библиотеки для доступа к буферу.
- **Для запуска:** библиотеки GTK4 (на Ubuntu 24.04 с GNOME уже есть).
- **SQLite отдельно НЕ нужен** — вкомпилён (`bundled`).
- `install.sh` проверяет наличие пакетов и подсказывает точную команду `apt install …`.

## 6.3 Офлайн/прокси-сборка — ✅ риск, проверяем на Этапе 0
Первая сборка тянет крейты из сети. Окружение с proxychains может мешать.
- **Проверяем в самом первом кодовом шаге (Этап 0).**
- Пользователь готов сам настроить прокси/скачать зависимости — на Этапе 0
  дадим точные команды (`cargo fetch` / настройка прокси / при нужде `cargo vendor`
  для сборки без сети).

## 6.4 Конфиг пользователя — ✅ после MVP, пока дефолты в коде
В MVP **конфиг-файла нет** — значения захардкожены (интервал 500 мс, 100 записей,
порог 1 МБ; клавиша задаётся в GNOME). Простой `~/.config/reclip/config.toml`
(интервал, размер истории) — первое улучшение после MVP.
