# 08. Технические решения (доп. вопросы)

Решения по группе 8 из `TODO.md` — вопросы, всплывшие по ходу проработки.

## 8.1 Синхронность vs async — ✅ синхронно, без tokio
Демон — простой цикл «поспал 500 мс → проверил буфер». Используем обычный
синхронный `std::thread::sleep`, **без tokio**. Проще и меньше зависимостей.
В пикере отдельно работает штатный главный цикл GTK/glib.

## 8.2 Логи и обработка ошибок — ✅ log + env_logger, anyhow
- Логи: крейты `log` + `env_logger`, вывод в stderr. systemd соберёт их в
  журнал — смотреть через `journalctl --user -u reclip`.
- Ошибки: `anyhow` (единый тип ошибок, удобно прокидывать).

## 8.3 Итоговый список зависимостей — ✅ утверждён
| Крейт | Зачем |

## 8.4 Пограничные состояния — ✅ договорились
- Первый запуск (базы ещё нет) → создаём базу и схему.
- Пустая история → пикер показывает «пусто».
- Демон не запущен, но вызвали `show` → окно всё равно открывается и показывает
  то, что есть в базе (новые копии просто не пишутся, пока демон не работает).
