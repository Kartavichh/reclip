# 04. Пикер (окно выбора по `Super+V`)

Решения по группе 4 из `TODO.md`.

## 4.1 Горячая клавиша — ✅ `Super+V` через настройку GNOME
На Wayland приложение не может само перехватить клавишу. Поэтому вешаем комбо
через **Настройки GNOME → Клавиатура → Свои комбинации** на команду `reclip show`.
- Комбинация по умолчанию: **`Super+V`** (как в Windows, обычно свободна).
- Дадим инструкцию и, по возможности, скрипт, который пропишет комбо автоматически.

## 4.2 Что делает выбор — ✅ только кладёт в буфер (авто-вставка позже)
Выбор записи **кладёт её в буфер обмена**; вставляет пользователь сам `Ctrl+V`.
Авто-вставка (эмуляция `Ctrl+V`) на Wayland сложна — требует `ydotool`/`uinput`
и отдельных прав. Отложена как самостоятельная фича после MVP.

## 4.3 Поведение окна — ✅ окно по центру, закрытие по Esc/фокусу/выбору
Обычное окно GTK4 по центру экрана. Закрывается:
- по `Esc`,
- при потере фокуса (клик мимо окна),
- сразу после выбора записи.

## 4.4 Навигация — ✅ стрелки + Enter + Esc + цифры; поиск после MVP
MVP: `↑`/`↓` — перемещение, `Enter` — выбрать, `Esc` — закрыть,
цифры `1–9` — быстрый выбор верхних записей.
**Поиск-фильтр по набору букв** — первое улучшение сразу после MVP (не в MVP).

## 4.5 Как выглядит — ✅ простой список строк-превью
MVP: вертикальный **список строк**, каждая запись — одна строка-превью.
Длинный/многострочный текст показываем укороченным (первая строка + «…»).
