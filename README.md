# reclip

Менеджер истории буфера обмена для Linux (GNOME/Wayland) — аналог `Win+V`.

Обычный буфер обмена помнит только последнее скопированное. `reclip` хранит
**историю** копирований и позволяет быстро вернуть любое из них по горячей
клавише `Super+V`.

## Возможности (MVP)

- Фоновый демон следит за буфером и копит историю **текста**.
- Окно по `Super+V`: список последних копирований, выбор возвращает запись в буфер.
- Навигация клавиатурой: `↑`/`↓` + `Enter`, цифры `1`–`9` для быстрого выбора, `Esc` — закрыть.
- Хранит последние 100 записей, дубликаты поднимает наверх, переживает перезагрузку.
- Один самодостаточный бинарь, база — встроенный SQLite (отдельно ставить не нужно).

## Требования

- Linux с GNOME на Wayland (разрабатывается и проверено на **Ubuntu 24.04**).
- Для сборки: `cargo` (Rust), `build-essential`, `libgtk-4-dev`.

```bash
sudo apt install build-essential libgtk-4-dev
# Rust, если ещё нет: https://rustup.rs
```

## Установка

```bash
git clone https://github.com/kartavich/reclip.git
cd reclip
./install.sh
```

Скрипт соберёт релиз, поставит бинарь в `~/.local/bin`, включит автозапуск
демона (systemd user-сервис) и напечатает, как назначить горячую клавишу.

**Назначьте `Super+V`** вручную (один раз):
Настройки → Клавиатура → Комбинации клавиш → Дополнительные комбинации → «+»

- Команда: `~/.local/bin/reclip show` (подставьте полный путь, например
  `/home/ВЫ/.local/bin/reclip show`)
- Клавиша: `Super+V`

## Использование

Копируйте как обычно (`Ctrl+C`) — история копится сама. Нажмите `Super+V`,
выберите запись стрелками/цифрой и `Enter` — она попадёт в буфер. Вставьте
`Ctrl+V`.

Команды:

```bash
reclip show     # открыть окно истории (то, что делает Super+V)
reclip list     # напечатать историю в терминал
reclip daemon   # запустить демон вручную (обычно им управляет systemd)
```

Управление сервисом:

```bash
systemctl --user status reclip.service      # статус демона
systemctl --user restart reclip.service     # перезапустить
journalctl --user -u reclip.service -f      # смотреть логи
```

## Как устроено

Один бинарь `reclip` с тремя под-командами и общей библиотекой:

- **`daemon`** — единственный, кто пишет в историю. GNOME/Wayland не присылает
  событий об изменении буфера, поэтому демон **опрашивает** его каждые 500 мс.
- **`show`** — окно-пикер на GTK4; только читает историю и кладёт выбор в буфер.
- **`list`** — печать истории в терминал (для отладки).

История — SQLite-файл `~/.local/share/reclip/history.db` (режим WAL для
одновременного доступа демона и пикера). Подробные проектные решения — в
[`docs/`](docs/), план и статус — в [`TODO.md`](TODO.md).

## Ограничения MVP

- Только **текст** (картинки — в планах).
- Окно открывается там, где его ставит композитор (на Wayland приложение не
  может задать свою позицию само).
- Пока нет: закрепления записей, поиска в окне, файла настроек, готовых
  бинарей в релизах, поддержки других окружений и Windows. См. backlog в `TODO.md`.

## Удаление

```bash
systemctl --user disable --now reclip.service
rm -f ~/.local/bin/reclip \
      ~/.config/systemd/user/reclip.service \
      ~/.local/share/reclip/history.db*
```
