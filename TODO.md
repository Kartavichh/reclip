# reclip — план и открытые вопросы

## 🚧 Глобальный план реализации (MVP)

Каждый этап — самостоятельный кусок: собирается и проверяется отдельно.
Порядок важен (каждый следующий опирается на предыдущий).
Статусы: ⬜ не начат · 🟨 в работе · ✅ готов

- ✅ **Этап 0 — Каркас + проверка риска.** (2026-07-07)
  Создан `Cargo.toml` (пока минимальный: `arboard`+`anyhow`, полный набор 8.3
  развернём на Этапе 1) и `src/main.rs` — проба чтения буфера.
  **Результат — оба риска сняты:** (а) `arboard` читает буфер на GNOME/Wayland
  через X11-бэкенд/XWayland — план Б (`wl-clipboard-rs`/`xclip`) не нужен;
  (б) `cargo` тянет крейты через proxychains без доп. настройки прокси.
  Раскладку модулей (7.1) разворачиваем на Этапе 1.

- ✅ **Этап 1 — Модель + хранилище.** (2026-07-07)
  `model.rs` (`ClipItem`, enum `Content`, превью-строка), `storage.rs`
  (SQLite WAL, схема + `PRAGMA user_version`, `add_text` с фильтрами 1.4/1.5,
  дедуп-подъём через «удалить+вставить» → свежий id наверху, обрезка до 100).
  Каркас CLI на `clap` (под-команды-заглушки). **11 юнит-тестов зелёные.**
  Зависимости этапа: clap, rusqlite(bundled), anyhow, directories.

- ✅ **Этап 2 — Абстракция буфера.** (2026-07-07)
  `clipboard.rs`: trait `Clipboard` (get_text/set_text, 7.2), реализация
  `ArboardClipboard` (пустой буфер → `Ok(None)`, не ошибка), плюс `MockClipboard`
  для тестов логики без дисплея. Проверено: мок-roundtrip в `cargo test` и
  живой roundtrip через `arboard` (`cargo test -- --ignored`) — оба зелёные.

- ⬜ **Этап 3 — Демон.**
  `daemon.rs` + под-команда `reclip daemon`: один экземпляр (`flock`, 3.3),
  цикл поллинга 500 мс (3.1), фильтры (1.4/1.5), запись в хранилище через
  правило дедупа (5.1). Логи `log`+`env_logger` (8.2).
  *Проверка: запускаем демон, копируем разное — история наполняется.*

- ⬜ **Этап 4 — CLI-просмотр (отладочный).**
  Под-команда `reclip list` — печатает историю в терминал.
  *Проверка: сквозной тест демон→база→list без GUI.*

- ⬜ **Этап 5 — Пикер (GUI).**
  `picker.rs` + под-команда `reclip show`: окно GTK4 по центру (4.3), список
  строк-превью (4.5), навигация стрелки/Enter/Esc/цифры (4.4), выбор кладёт в
  буфер (4.2), закрытие по Esc/фокусу/выбору. Пикер только читает базу (5.1).
  *Проверка: `reclip show` открывает окно, выбор возвращает запись в буфер.*

- ⬜ **Этап 6 — Запуск и установка.**
  systemd user-сервис `reclip.service` (3.2), `install.sh` (6.1) с проверкой
  зависимостей (6.2), инструкция по `Super+V` в GNOME (4.1).
  *Проверка: после install.sh демон стартует сам, Super+V открывает пикер.*

- ⬜ **Этап 7 — Полировка MVP.**
  Сквозной ручной тест (пункт 11), обновление `README`, мелкие правки.
  *Проверка: полный сценарий «скопировал → Super+V → выбрал → вставил» работает.*

# Открытые вопросы (все решены ✅)

> Рабочее имя проекта — `reclip` (см. вопрос 0). Менеджер истории буфера обмена
> для Linux (GNOME/Wayland), аналог `Win+V`. Язык — Rust, GUI — GTK4.
> Этот файл — список того, что надо решить **до** написания кода.
> Каждый вопрос обсуждаем отдельно, решение переносим в `docs/`.

Статусы: ⬜ не обсуждали · 🟨 обсуждаем · ✅ решено (ответ в `docs/`)

## 1. Что именно ловим и храним (модель данных)
- ✅ **1.1 Типы содержимого в MVP.** Только текст; в модели заложен enum типа. → `docs/01-model.md`
- ✅ **1.2 Две «выделения» X11/Wayland.** Только CLIPBOARD; PRIMARY отложен. → `docs/01-model.md`
- ✅ **1.3 Дубликаты.** Существующую запись поднимаем наверх. → `docs/01-model.md`
- ✅ **1.4 Пустое/мусорное.** Игнорируем пустое и строки из одних пробелов. → `docs/01-model.md`
- ✅ **1.5 Лимит размера одной записи.** Порог ~1 МБ, сверх — не сохраняем. → `docs/01-model.md`
- ✅ **1.6 Сколько записей храним.** По умолчанию 100, вытесняем старые (FIFO). → `docs/01-model.md`

## 2. Хранилище истории
- ✅ **2.1 Где лежит файл.** `~/.local/share/reclip/history.db`. → `docs/02-storage.md`
- ✅ **2.2 Формат файла.** SQLite (`rusqlite`, bundled) — один бинарь. → `docs/02-storage.md`
- ✅ **2.3 Живучесть между перезагрузками.** Да, читаем базу при старте. → `docs/02-storage.md`
- ✅ **2.4 Одновременный доступ.** SQLite WAL разруливает сам. → `docs/02-storage.md`
- ✅ **2.5 Версионирование формата.** `PRAGMA user_version` + миграции. → `docs/02-storage.md`

## 3. Демон (фоновое слежение)
- ✅ **3.1 Интервал поллинга.** 500 мс по умолчанию. → `docs/03-daemon.md`
- ✅ **3.2 Как запускается.** Вручную (разработка) + systemd user-сервис (боевой). → `docs/03-daemon.md`
- ✅ **3.3 Один экземпляр.** `flock` на `$XDG_RUNTIME_DIR/reclip.lock`. → `docs/03-daemon.md`
- ✅ **3.4 Способ чтения буфера.** `arboard` за trait + ранняя проверка на GNOME, план Б. → `docs/03-daemon.md`

## 4. Пикер (окно выбора по `Super+V`)
